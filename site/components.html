Feo.css - Components

A limited set of components (mostly based on basic HTML elements, but are more complex in their implementation).

Tables

This is how You implement it. Note: the additional div is required to ensure the table is sidescrollable and does not break layouts on small screens. 

<div>
  <table>
    <thead>
      <tr>
        <th></th><th>...</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td></td><td>...</td>
      </tr>
    </tbody>
  </table>
</div>

There are several custom properties available that can be overwritten to control the looks of the table.

Custom property Selector Description
--table-radius table Sets the radius of the corners
--table-color table Sets the color of the borders and header

Form elements

Consistent styling is available for the most common form elements: input fields, text-areas, and two kinds of dropdowns. In addition, various states like :hover and :disabled are covered.

All the fields in the form follow a specific implementation for the styling. 

<label>
  Field label
  <[element] ... />
</label>

Switch

A switch is a specific component that can be used as a solitary checkbox, or for something else.

The implementation is a slightly different from the form elements, as you will put the label text after the switch in most cases. 

<label>
  <input type="checkbox" ... />
  Switch label
</label>
Custom property Selector Description
--switch-color input[type="checkbox"] Sets the background color of the switch on :checked. Default is green.
