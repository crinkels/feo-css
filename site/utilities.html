Feo.css - Utilities

There is nothing wrong with utility-first CSS. It only becomes a problem when it is utility-only. If you looked at the layouts layer, utilities there are used to control layout settings. It is an incredible scalable setup.

Feo.css offers a very limited set of utility classes, mostly based on the available design tokens. In all the classes that you see, the <z> is a placeholder for the number corresponding to the correct design token.

Margin

Class Design token CSS property
m-<z> --token-size-<z> margin
mt-<z> --token-size-<z> margin-top
mr-<z> --token-size-<z> margin-right
mb-<z> --token-size-<z> margin-bottom
ml-<z> --token-size-<z> margin-left

Padding

Class Design token CSS property
p-<z> --token-size-<z> padding
pt-<z> --token-size-<z> padding-top
pr-<z> --token-size-<z> padding-right
pb-<z> --token-size-<z> padding-bottom
pl-<z> --token-size-<z> padding-left

Typography

Class Design token CSS property
size-<z> --token-size-<z> font-size
monospace/serif/sans-serif - font-family
bold/regular/italc/uppercase - Various
text-left/right/center - text-align

maxw-<z>

Based on the --token-bp-<z> tokens, control the max-width property.

click-area

Makes the entire element with this class clickable, based on the first <a /> it can find in its (sub-)DOM. It does not have to be a direct descender.
